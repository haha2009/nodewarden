# 新登录流程深度测试计划

## 前置
- 启动 `npm run dev`（Wrangler 端口 8787），确保已有登录用户
- 浏览器硬刷新（Cmd+Shift+R）
- 所有测试基于当前 VaultEditor.tsx 的最新代码

---

## 一、新建登录 — 默认状态

| # | 操作 | 预期 |
|---|------|------|
| 1.1 | 进入"新增"→ 选择"登录" | 看到"网站主页"输入框 + "登录页面"输入框（带 + 添加网站 按钮） |
| 1.2 | 什么都不填，检查页面 | 两个输入框为空，+ 添加网站 按钮可见 |
| 1.3 | 点"保存" | 应成功创建（URI 空值在前端 encryptUris 过滤掉），不报错 |

## 二、新建登录 — 填写完整

| # | 操作 | 预期 |
|---|------|------|
| 2.1 | 网站主页输入 `https://example.com` | 输入框正常显示，AI Sparkles 按钮变为可点 |
| 2.2 | 登录页面输入 `https://example.com/login` | 输入框正常显示，+ 添加网站 按钮右侧 |
| 2.3 | 填写用户名/密码/TOTP | 全部正常 |
| 2.4 | 点"保存" | 创建成功 |
| 2.5 | 在列表中找到刚创建的条目，点开查看 | 详情页应显示两条 URI（网站主页 + 登录页面），均有"打开""复制"按钮 |

## 三、新建登录 — + 添加网站（多网站）

| # | 操作 | 预期 |
|---|------|------|
| 3.1 | 继续在 2 的基础上编辑 | 主页/登录页预填正确 |
| 3.2 | 点 + 添加网站 | 下方出现一组新行："网站主页" + "登录页面" + "✕ 删除" |
| 3.3 | 新主页输入 `https://other.com`，新登录页输入 `https://other.com/login` | 正常输入 |
| 3.4 | 点"保存" → 重新打开编辑 | 两组网站各 4 条 URI 全部显示，主页/登录页配对正确 |
| 3.5 | 点某组的 ✕ 删除 | 该组整对（主页+登录页 2 条）全部移除 |

## 四、编辑已有 — 旧数据兼容

| # | 操作 | 预期 |
|---|------|------|
| 4.1 | 找一个只有 1 条 URI 的老条目，点编辑 | 主页输入框预填该 URI，登录页输入框为空 |
| 4.2 | 给登录页输入一个 URL + 保存 | 保存成功 |
| 4.3 | 重新打开编辑 | 主页+登录页均显示之前输入的内容 |
| 4.4 | 找一个 2 条 URI 的老条目（来自旧 match 选择器） | 主页框显示 match=null 的 URI，登录页框显示 match=1 的 URI |

## 五、AI 自动填充

| # | 操作 | 预期 |
|---|------|------|
| 5.1 | 主页输入 `https://github.com/claw` | Sparkles 按钮亮起 |
| 5.2 | 点击 Sparkles 按钮 | 名称字段自动填入"GitHub"，按钮短暂闪蓝 |

## 六、边界情况

| # | 操作 | 预期 |
|---|------|------|
| 6.1 | 主页填 `https://a.com`，登录页填相同 URL | 保存后详情页只显示 1 条（encryptUris 去重） |
| 6.2 | 主页填空白，登录页填 URL | 主页为空时不影响保存，登录页的 URI 正常存储 |
| 6.3 | + 添加网站后不填任何 URL，直接保存 | 空 URI 被过滤，保存不报错 |
| 6.4 | 输入超长 URL（>10000 字符） | 服务端校验拦截，返回验证错误 |
| 6.5 | 输入带中文/特殊字符的 URL | 正常保存，详情页正常显示 |

|---|------|------|
| 8.1 | 主页输入框填 `<script>alert(1)</script>` | 保存后详情页显示为纯文本，不会执行脚本 |

- dev server 跑在 `http://localhost:8787`（Wrangler），不是 5173
